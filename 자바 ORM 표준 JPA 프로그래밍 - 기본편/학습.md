# SQL 중심적인 개발의 문제점

객체 지향 - 관계형 DB

객체 -> 관계형 DB에 보관 / 관리

SQL 중심적인 개발의 문제점
- 쿼리가 너무 많다
- 객체 필드를 추가하면 쿼리문을 다 고쳐야한다
- SQL에 의존적인 개발

하지만 우리는 객체를 관계형 데이터베이스에 저장해야 한다</br>
마땅한 대안이 없다

## 객체와 관계형 데이터 베이스의 차이

### 상속
슈퍼타입-서브타입 관계

계속 조인 쿼리로 가져와야 한다

### 연관 관계

객체는 참조를 사용: member.getTeam()

테이블은 외래 키를 사용: JOIN ON M.TEAM_ID = T.TEAM_ID
